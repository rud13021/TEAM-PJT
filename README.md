# 공공데이터 기반 지역 정보 공유 커뮤니티

> "우리 어디서 봐?" 소모적인 약속 장소 정하기와 정보 탐색을 한 번에 해결하는 중간 지점 매칭 및 로컬 큐레이션 서비스

본 프로젝트는 여러 출발지의 중간 지점을 도출하는 동선 최적화 알고리즘을 바탕으로, 해당 지역의 축제, 맛집, 쇼핑 등 다양한 로컬 정보를 공공데이터 기반으로 추천 및 공유하는 웹 애플리케이션입니다.

--------------------------------------------------------------

## 프로젝트 소개

- 개발 기간: 3일 (단기 집중 프로젝트)
- 배포 주소: [Netlify 배포 링크 입력] (예: https://banmanwa1.netlify.app)
- 주요 타깃 서울 전역에서 모이는 모든 모임 구성원
- 핵심 목표 오프라인 만남의 시작(동선 최적화)부터 끝(로컬 정보 및 코스 설계)까지 원스톱 서비스 제공

--------------------------------------------------------------

## 기술 스택 및 개발 환경

### Development Environment
- OS Windows 11
- IDE Visual Studio Code

### Tech Stack
- Frontend Vue 3 (Composition API), Vite, Pinia (전역 상태 관리)
- Language HTML5, CSS3, JavaScript
- Deployment Netlify

### Integrations & APIs
- Map & Mobility Kakao Map API, Kakao Mobility API
- Data TourAPI, OpenAPI (공공데이터 포털)
- AI OpenAI API (맞춤형 추천 챗봇)
- Search Naver Blog Search API

## 핵심 기능 (Features)

### 1. 동선 최적화 및 중간 지점 도출 (Main)
- 최대 5명의 출발지를 입력받아 모두에게 공평한 최적의 중간역 후보 3곳 도출
- 각 후보지까지의 도보 시간, 이동 거리 계산을 통한 합리적 선택 지원

### 2. 로컬 정보 큐레이션 및 상세 동선 시각화 (Map & Recommend)
- 출발지부터 최종 중간지점까지의 동선을 직관적인 라인으로 지도 위에 매핑
- 관광지, 쇼핑, 축제 등 7대 카테고리 필터를 통해 중간지점 주변 추천 장소 마커 표시
- 동선 낭비를 최소화하는 최단 이동 여행 코스 추천 및 상세 이동 거리/소요 시간 비교

### 3. 나만의 보관소 및 공유 (Favorites)
- 마음에 드는 목적지나 코스를 담아두는 개인 저장소 기능 제공
- 카카오톡 공유 API 연동을 통해 모임원들에게 실시간 코스 전송 지원

### 4. 실시간 소통 게시판 (Board)
- 별도의 가입 절차 없이 후기와 꿀팁을 나누는 익명 게시판 (비밀번호 기반 CRUD)
- 댓글 및 추천 기능 포함

### 5. AI 맞춤형 코스 추천 챗봇 (Chat)
- OpenAI API 연동으로 기본 추천 외에 사용자 취향을 반영한 실시간 맞춤형 대안 코스 설계

### 6. 로컬 큐레이션 강화 (Blog & Calendar)
- 중간 지점 결과에 매핑된 실제 네이버 블로그 방문 후기 링크 자동 제공
- 날짜별 축제 및 공연 정보를 직관적으로 파악할 수 있는 축제 캘린더 제공

## 프로젝트 구조 (Directory Structure)

```text
team-pjt/
├── public/                 # 빌드 프로세스에서 제외되는 정적 자원 폴더
│   └── data/               # 가상 데이터나 초기 설정값용 로컬 JSON 파일들
├── scripts/                # 빌드, 배포, 자동화 등에 쓰이는 커스텀 스크립트 파일들
├── src/                    # 실제 애플리케이션 소스 코드가 위치하는 핵심 폴더
│   ├── components/         # 화면에 렌더링될 재사용 가능한 UI 컴포넌트들
│   │   ├── board/          # 익명 게시판 관련 컴포넌트
│   │   ├── common/         # 공통 컴포넌트 (버튼, 모달, 인풋 등)
│   │   ├── group/          # 모임/그룹 기능 관련 컴포넌트
│   │   ├── home/           # 홈(메인) 화면 전용 컴포넌트
│   │   ├── icon/           # SVG 등 아이콘 관련 컴포넌트
│   │   ├── map/            # 중간 지점 추천 컴포넌트
│   │   └── recommend/      # 선택 목적지 추천 여행 코스 관련 컴포넌트
│   ├── composables/        # 공통 비즈니스 로직을 분리한 커스텀 훅 (Composition API)
│   ├── router/             # 페이지 이동 및 라우팅을 정의하는 설정 파일들
│   ├── services/           # 외부 백엔드 API와의 비동기 통신을 담당하는 파일들 (Axios 등)
│   ├── stores/             # 전역 상태 관리 파일들 (Pinia 스토어)
│   ├── utils/              # 계산 관련 로직 파일들
│   ├── views/              # 라우터와 매핑되는 페이지 단위의 최상위 컴포넌트 화면들
│   ├── App.vue             # 최상위 루트(Root) 싱글 파일 컴포넌트
│   ├── main.js             # 애플리케이션 인스턴스 생성 및 라이브러리 등록 시작점
│   └── style.css           # 프로젝트 전역에 적용되는 공통 CSS 스타일시트
├── .env                    # 외부 노출을 제한해야 하는 환경 변수 설정 파일 (API 주소 등)
├── .gitignore              # Git 버전 관리에서 제외할 파일/폴더 지정 목록
├── index.html              # 브라우저가 읽는 진입점 HTML 문서 (Vite 구조의 핵심)
├── package.json            # 라이브러리 의존성(Dependencies) 및 실행 스크립트 명세서
└── vite.config.js          # 빌드 도구인 Vite의 세부 플러그인 및 설정 파일
